## 21. What is the /etc/shadow file?

**Explanation:**
- The `/etc/shadow` file contains **encrypted passwords** and password aging information for users.
- Only the **root user** can read or modify it.

**Example Line:**
```

ec2-user:$6$abc$Qer...:19145:0:99999:7:::

````
Each field stores information like last password change, expiry, and inactivity period.

## 23. What are Linux runlevels?

**Explanation:**
Runlevels define the **mode of operation** for the system.

| Runlevel | Mode                        |
| -------- | --------------------------- |
| 0        | Halt / Shutdown             |
| 1        | Single user mode            |
| 2        | Multiuser (no networking)   |
| 3        | Multiuser (with networking) |
| 4        | Unused / Custom             |
| 5        | Graphical mode              |
| 6        | Reboot                      |

**Command to check current runlevel:**

```bash
runlevel
```

## 29. What is the difference between find and locate?

| Feature           | find                     | locate                  |
| ----------------- | ------------------------ | ----------------------- |
| Real-time         | Searches live filesystem | Uses pre-built database |
| Speed             | Slower                   | Faster                  |
| Requires updatedb | No                       | Yes                     |
| Example           | `find / -name file.txt`  | `locate file.txt`       |

**Tip:**
Run `updatedb` before using `locate` to refresh the index.

## 32. How do you change a process’s priority in Linux?

**Command:**

```bash
nice -n [value] command
renice [value] -p [PID]
```

**Explanation:**

* `nice` → sets priority when starting a process
* `renice` → changes priority of a running process
* Lower nice value = higher priority

**Example:**

```bash
nice -n 10 tar -czf backup.tar.gz /data
renice -5 -p 2345
```

## 40. What is the dmesg command used for?

**Explanation:**

* Displays **kernel ring buffer messages** (mainly hardware and boot info).

**Command:**

```bash
dmesg | less
```

**Use Case:**

* To troubleshoot **hardware errors**, **driver issues**, or **boot-time events**.

**Example:**

```bash
dmesg | grep eth0
```

→ Shows messages related to network interface `eth0`.
