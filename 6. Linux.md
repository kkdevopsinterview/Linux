## 81. What is the purpose of the /etc/hosts file?

**Explanation:**
- The `/etc/hosts` file maps **hostnames to IP addresses**.
- It’s checked before DNS when resolving names.

**Example:**
```

127.0.0.1   localhost
192.168.1.10   webserver

````

**Use Case:**  
Useful for local DNS overrides or private network configurations.

## 88. How do you create a new systemd service in Linux?

**Steps:**

1. Create service file:

   ```bash
   sudo vi /etc/systemd/system/myscript.service
   ```
2. Add content:

   ```
   [Unit]
   Description=My Custom Script Service

   [Service]
   ExecStart=/usr/local/bin/myscript.sh
   Restart=always

   [Install]
   WantedBy=multi-user.target
   ```
3. Reload and enable service:

   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable myscript
   sudo systemctl start myscript
   ```

**Check status:**

```bash
systemctl status myscript
```

## 91. How do you secure an SSH server?

**Steps:**

1. **Disable root login:**

   ```bash
   vi /etc/ssh/sshd_config
   PermitRootLogin no
   ```
2. **Change default port:**

   ```
   Port 2222
   ```
3. **Allow specific users:**

   ```
   AllowUsers devops
   ```
4. **Restart SSH:**

   ```bash
   systemctl restart sshd
   ```
5. **Use key-based authentication** instead of passwords.

## 92. What is SELinux, and how does it work?

**Explanation:**

* **SELinux (Security-Enhanced Linux)** provides **mandatory access control (MAC)**.
* Controls how applications and users can access files and resources.

**Commands:**

```bash
getenforce       # Check mode
setenforce 0     # Temporarily disable
```

**Modes:**

* **Enforcing:** Policies are enforced
* **Permissive:** Logs violations only
* **Disabled:** SELinux off

**Config File:** `/etc/selinux/config`

## 95. How do you configure a static IP in Linux?

**For RHEL/CentOS:**
Edit the interface file:

```bash
vi /etc/sysconfig/network-scripts/ifcfg-eth0
```

Add:

```
BOOTPROTO=none
IPADDR=192.168.1.10
NETMASK=255.255.255.0
GATEWAY=192.168.1.1
DNS1=8.8.8.8
ONBOOT=yes
```

**Apply:**

```bash
systemctl restart network
```

**For Ubuntu:**
Edit `/etc/netplan/*.yaml` and apply with `netplan apply`.

## 96. How do you enable IP forwarding in Linux?

**Command:**

```bash
echo 1 > /proc/sys/net/ipv4/ip_forward
```

**To make permanent:**
Edit `/etc/sysctl.conf`:

```
net.ipv4.ip_forward = 1
```

**Apply:**

```bash
sysctl -p
```

**Use Case:**
Required for routing or NAT configuration.

## 97. What is a kernel panic, and how do you troubleshoot it?

**Explanation:**

* A **kernel panic** occurs when the Linux kernel encounters a fatal error it cannot recover from.

**Troubleshooting Steps:**

1. Check logs:

   ```bash
   cat /var/log/messages | grep -i panic
   dmesg | tail
   ```
2. Boot using **rescue mode** or **previous kernel**.
3. Check **hardware**, **drivers**, and **recent kernel updates**.
4. Use:

   ```bash
   journalctl -xb
   ```

**Tip:**
Avoid modifying `/boot` or kernel-related packages without backups.

## 99. How do you perform a filesystem check in Linux?

**Commands:**

```bash
fsck /dev/sda1
```

**Explanation:**

* `fsck` checks and repairs filesystem inconsistencies.
* Should be run when the disk is **unmounted**.

**Example:**

```bash
umount /dev/sda1
fsck -y /dev/sda1
```

**Tip:**
Use cautiously — incorrect use can lead to data loss.
