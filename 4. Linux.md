## 41. What is the difference between ps, top, and htop?

| Command | Description |
|----------|--------------|
| `ps` | Displays snapshot of running processes |
| `top` | Real-time view of system processes and usage |
| `htop` | Enhanced, colorful version of `top` with interactive features |

**Examples:**
```bash
ps aux
top
htop
````

**Tip:**
Use `htop` to sort processes interactively by CPU or memory.

## 42. What are the different types of process states in Linux?

| State | Description                                        |
| ----- | -------------------------------------------------- |
| R     | Running or runnable (on CPU or waiting for CPU)    |
| S     | Sleeping (waiting for event)                       |
| D     | Uninterruptible sleep (usually I/O)                |
| T     | Stopped or traced                                  |
| Z     | Zombie (process completed but parent not informed) |

**Check process states:**

```bash
ps -eo pid,stat,comm
```

## 46. How do you mount and unmount a filesystem?

**Commands:**

```bash
mount /dev/sdb1 /mnt
umount /mnt
```

**Steps:**

1. Create mount point: `mkdir /mnt/data`
2. Mount: `mount /dev/xvdb1 /mnt/data`
3. Verify: `df -h`
4. Unmount: `umount /mnt/data`

**Permanent Mount:** Add to `/etc/fstab`.

## 47. What is LVM (Logical Volume Manager) in Linux?

**Explanation:**
LVM allows flexible disk management — resizing, adding, or removing storage without downtime.

**Components:**

* **PV (Physical Volume):** Actual disks
* **VG (Volume Group):** Pool of PVs
* **LV (Logical Volume):** Mounted storage from VG

**Commands:**

```bash
pvcreate /dev/sdb
vgcreate vg_data /dev/sdb
lvcreate -L 10G -n lv_backup vg_data
mkfs.ext4 /dev/vg_data/lv_backup
mount /dev/vg_data/lv_backup /backup
```

## 50. What is the difference between hard links and symbolic links?

| Type                      | Description                           | Command             |
| ------------------------- | ------------------------------------- | ------------------- |
| Hard Link                 | Points to same inode as original file | `ln file1 file2`    |
| Symbolic Link (Soft Link) | Shortcut pointing to file path        | `ln -s file1 link1` |

**Example:**

```bash
ln file1.txt file2.txt      # Hard link
ln -s file1.txt link1.txt   # Soft link
```

**Note:**
Deleting the original file breaks soft link but not hard link.

## 53. What is the difference between TCP and UDP?

| Feature     | TCP                     | UDP            |
| ----------- | ----------------------- | -------------- |
| Type        | Connection-oriented     | Connectionless |
| Reliability | Reliable (acknowledged) | Unreliable     |
| Speed       | Slower                  | Faster         |

## 56. What is iptables, and how is it used in Linux?

**Explanation:**

* `iptables` is a **firewall utility** used to configure packet filtering rules.

**Common Commands:**

```bash
iptables -L                # List rules
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
iptables -A INPUT -p tcp --dport 80 -j DROP
```

**Example:**
Allow SSH (22) and block HTTP (80).

**Tip:**
For newer systems, `firewalld` or `nftables` is used.

* Enables login without password
